USINGZ

Delphi{$DEFINE USINGZ}

C++#define USINGZ

C# #define USINGZ


With regard to clipping solutions, occasionally users may need to identify vertices that have been created at path intersections and even assign user-defined data for these vertices. To facilitate this, in addition to the X and Y members in Point64 and PointD structures, the pre-processor directive USINGZ will add an Int64 Z member to these structures.

This Z member is always of type Int64, even when the X and Y members are type double as in the PointD structure. And it's not a coordinate since it will only contain user-defined data. With the USINGZ directive set, users can assign user-defined data to each Point64 or PointD in both subject and clip paths prior to clipping. And these Z values will be retained if these vertices are returned in clipping solutions. Also, a ZCallback function can be assigned to Clipper64 and ClipperD objects which allows users to assign user-defined Z values when new vertices are created at path intersections.

There are C++ and C# examples of using USINGZ in their respective Examples folders.

Note: USINGZ will have a small impact on performance, so the library's default is to leave this directive undefined.
